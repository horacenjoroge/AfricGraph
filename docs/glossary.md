# Glossary

Terms and concepts used in AfricGraph documentation.

## A

**ABAC (Attribute-Based Access Control)**
A security model that grants access based on attributes of the subject, resource, and environment.

**Alert**
A notification triggered when specific conditions are met (e.g., high risk score, fraud pattern detected).

**Anomaly Detection**
Machine learning technique to identify unusual patterns or outliers in data.

## B

**Backup**
A copy of data stored for disaster recovery purposes.

**Business Entity**
A company or organization in the system.

**Burn Rate**
The rate at which a business consumes cash, typically measured monthly.

## C

**Cache**
Temporary storage for frequently accessed data to improve performance.

**Cypher**
The query language used by Neo4j graph database.

**Composite Risk Score**
An overall risk score calculated from multiple risk factors.

## D

**DataLoader**
A pattern used in GraphQL to batch and cache data requests, preventing N+1 query problems.

**Disaster Recovery**
Procedures to restore systems after a failure or disaster.

## E

**Elasticsearch**
Search engine used for full-text search and indexing.

**Entity**
A node in the graph database (e.g., Business, Person, Transaction).

## F

**Factor Score**
An individual risk factor score (e.g., payment behavior, supplier concentration).

**Fraud Pattern**
A suspicious pattern detected in transaction or relationship data.

**Full-Text Search**
Search that matches text content, not just exact matches.

## G

**Graph Database**
A database that stores data as nodes and relationships (Neo4j).

**GraphQL**
A query language for APIs that allows clients to request exactly the data they need.

**Graph Traversal**
Navigating through relationships in a graph database.

## H

**Health Check**
An endpoint or script that verifies system components are functioning.

**HHI (Herfindahl-Hirschman Index)**
A measure of market concentration, used for supplier concentration analysis.

## I

**Incremental Backup**
A backup that only includes changes since the last backup.

**Integration Test**
A test that verifies multiple components work together.

## J

**JWT (JSON Web Token)**
A token-based authentication mechanism.

## K

**Knowledge Graph**
A graph database representing real-world entities and their relationships.

## L

**Load Testing**
Testing system performance under expected load conditions.

**Locust**
A load testing tool used for performance testing.

## M

**ML Model**
Machine learning model used for predictions (e.g., credit scoring).

**Middleware**
Software that processes requests before they reach the main application.

## N

**Neo4j**
The graph database used by AfricGraph.

**Node**
An entity in the graph database (e.g., Business, Person).

## O

**Ontology**
A formal representation of knowledge as a set of concepts and relationships.

**OpenAPI**
A specification for describing REST APIs (formerly Swagger).

**OWASP**
Open Web Application Security Project - security best practices.

## P

**Path**
A sequence of nodes and relationships connecting two entities.

**Permission**
Authorization to perform an action on a resource.

**Prometheus**
Monitoring and alerting toolkit.

## Q

**Query**
A request for data from a database or API.

## R

**Relationship**
A connection between two nodes in the graph database.

**Retention Policy**
Rules for how long backups are kept before deletion.

**Risk Score**
A numerical value (0-100) indicating the risk level of a business.

**RTO (Recovery Time Objective)**
The maximum acceptable time to restore service after a failure.

**RPO (Recovery Point Objective)**
The maximum acceptable amount of data loss measured in time.

## S

**Schema**
The structure of data in a database or API.

**Shell Company**
A business entity with no real operations, often used for fraud.

**Subgraph**
A portion of the full graph containing specific nodes and relationships.

## T

**Transaction**
A financial transaction between entities.

**TTL (Time To Live)**
The duration data is cached before expiration.

## U

**Unit Test**
A test that verifies a single function or component works correctly.

## V

**VPS (Virtual Private Server)**
A virtualized server used for hosting applications.

## W

**Workflow**
A multi-step process with approvals and conditions.

**Workflow Instance**
A running instance of a workflow definition.

## X

**XSS (Cross-Site Scripting)**
A security vulnerability where malicious scripts are injected into web pages.

## Y

(No terms starting with Y)

## Z

**Zero-Downtime Deployment**
A deployment strategy that maintains service availability during updates.
